---
プロジェクトで定義されている CI チェックを実行し、今回の変更に起因する問題があれば修正する。

## 実行フロー

1. `AGENTS.md`、`CLAUDE.md`、README、GitHub Actions、package scripts、Makefile、言語固有のビルド設定を確認し、このリポジトリの正式な検証コマンドを特定する。
2. 現在の差分と CI 構成から、変更に関係する lint、format、type check、test、build、生成物チェックを選ぶ。特定の package manager やツールを前提にしない。
3. 依存関係の軽いチェックから順に実行する。monorepo では、プロジェクトの既存コマンドが許す範囲で変更対象にスコープする。
4. 失敗した場合はログと関連コードを調査し、今回の変更に起因する問題だけを最小限の差分で修正する。既存の失敗は修正せず、区別して報告する。
5. 修正したチェックを再実行し、その後に必要な関連チェックを実行する。選んだチェックがすべて成功するまで繰り返す。

最後に、実行したコマンド、成功・失敗、修正内容、未解決の既存問題を簡潔に報告する。
