---
name: suggest-skill
description: セッションの会話の流れから再利用可能なagent skillを提案する
---
現在のセッションの会話フローを振り返り、Codex CLIまたはClaude Codeで繰り返し使えるskillとして切り出せるパターンがないか分析・提案する。

## 実行フロー

### 1. セッションの振り返り

現在のセッションで行った作業を時系列で整理する:

- 実行したスキル（slash command）の順序
- 手動で行った操作（ツール呼び出し、コマンド実行）
- スキル間の手動ブリッジ（あるスキルの出力を次の操作の入力にした箇所）
- ユーザーが追加で指示した操作（「issueも作って」「リンクも追記して」等）

### 2. パターンの特定

以下の観点でスキル化すべきパターンを探す:

**スキル化の候補**:
- スキルAの後に毎回手動で行う定型操作 → 独立スキルまたはスキルAへの統合
- 複数ツールを組み合わせた一連の手動操作 → 新規スキル
- ユーザーが「〇〇もやって」と追加指示した操作 → 次回から自動化

**スキル化しないもの**:
- 1回限りの特殊対応
- 判断の余地が大きく定型化できない操作
- 既存スキルで十分カバーされている操作

### 3. 既存スキルとの関係整理

このpluginに同梱された `skills/` と、実行環境から参照できるproject-local / global skillの一覧を確認し、提案するskillが:

- 既存スキルと重複しないか
- 既存スキルのオプションとして追加すべきか
- 既存スキルのパイプラインの一部として組み込むべきか

を確認する。

### 4. 提案の出力

各提案について以下の形式で出力する:

```
## 提案 N: /スキル名

### 概要
1-2行でスキルの目的を説明

### トリガー
どのような場面で使うか（「/reply-reviews の後に」「PRマージ前に」等）

### 入力 → 出力
- 入力: 何を受け取るか
- 出力: 何を生成・実行するか

### 今回のセッションでの該当箇所
セッション内のどの操作がこのスキルに対応するか具体的に示す

### 既存スキルとの関係
新規 / 既存スキルの拡張 / パイプライン統合

### 実装の方向性
スキルの主要ステップを箇条書きで示す（詳細な実装はしない）
```

### 5. ユーザーとの擦り合わせ

提案一覧を提示し、以下を確認する:

- 作成するスキルの選択
- 配布plugin、global skill、project-local skillのどこへ置くか
- スキル名の調整
- フローの修正点

ユーザーが承認したスキルのみ実装に進む。

## 制約事項

- 提案のみ行い、承認なしにスキルファイルを作成しない
- 既存スキルとの重複を必ずチェックする
- 「便利かもしれない」程度の弱い根拠では提案しない。セッション内で実際に行った操作に基づく提案のみ
